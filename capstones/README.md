# Chapter 17 -- Capstone Projects

Pick *one* of the three capstones below. Work on it for four to six
weeks. You may work alone or in a team of two or three. By the end,
submit three artifacts: a working code repository on GitHub, a short
IEEE-style report (about eight pages), and a ten-minute live demo
with a viva.

Choose a capstone that matches your level. Capstone 1 is friendly
for UG students. Capstone 2 suits UG or PG students. Capstone 3 is
best for PG students or strong UG teams.

Before you start, re-read Chapters 1-16 and list the patterns you
will reuse. A good capstone borrows ideas; it does not invent
everything from scratch.

## Projects

- [`1-smart-study-assistant/`](1-smart-study-assistant/) -- an agentic-RAG study assistant over textbook PDFs, with quiz generation and spaced repetition. Includes the book's starter scaffold and skeleton `app.py`.
- [`2-multi-agent-research-bot/`](2-multi-agent-research-bot/) -- a planner-researcher-writer-citer-editor pipeline producing a cited Markdown report. README-only; no starter code is given in the book.
- [`3-coding-agent-hitl/`](3-coding-agent-hitl/) -- a SWE-bench-lite style agent that fixes Python bugs with human-approval gates. README-only; no starter code is given in the book.

Each project folder has its own README with the objective, chapter
map, architecture, functional requirements, evaluation rubric, and
submission checklist, all taken directly from the book.
